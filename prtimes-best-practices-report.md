## 1. PR TIMES公式ガイドライン・推奨フォーマット（最新版）

PR TIMES MAGAZINEが公開している公式ガイドラインに基づき、プレスリリース作成の基本ルールと掲載確率を高める12のコツを整理する。

### 1-1. 基本の5つの構成要素

プレスリリースは以下の5要素で構成される。これはPR TIMESが公式に推奨する「型」であり、この構成に沿って書くことでメディア関係者に伝わりやすい文書となる。

1. **タイトル・見出し** -- プレスリリースが読まれるかどうかを左右する最重要要素
2. **リード文（要約文）** -- 全体の要約。タイトルの次に読まれ、本文を読むかの判断材料となる
3. **本文** -- 発表内容の核。十分な情報量と読みやすさの共存が求められる
4. **画像** -- 1リリースあたり3〜5枚以上が理想。トップ画像はSNSシェア時のサムネイルになる
5. **連絡先・問い合わせ先** -- 社名・担当部署・担当者名・住所・電話番号・メールアドレスを必ず記載

### 1-2. 掲載確率を上げる12のコツ（PR TIMES MAGAZINE公式）

| No. | コツ | 詳細 |
|-----|------|------|
| 1 | タイトルは30文字以内で結論を伝える | 1行あたり30文字程度。キーワード最大3つ。形容詞でなく数値。感嘆符多用禁止 |
| 2 | 5W2Hを押さえたリード文を作成する | 250〜300字が理想、最大400字以内。Who/What/Where/When/Why/How/How much＋展望 |
| 3 | 基本の構成・流れを意識した本文を作成する | 「三角形型」＝タイトル→リード文→本文と情報量を増やしていく。逆三角形（結論先行）で書く |
| 4 | 社会性・トレンド性を取り入れる | 「なぜ今この情報を発表するのか」の理由付け。メディアフックを意識 |
| 5 | 数値・データで裏付ける | 曖昧な表現を避け、具体的な数字で説得力を出す |
| 6 | 画像を3〜5枚以上掲載する | 商品写真、図解、代表者写真、利用シーンなど多角的に |
| 7 | 専門用語には注釈をつける | 業界外の読み手にも理解できるように |
| 8 | 箇条書き・小見出しで視認性を高める | 流し読みでも要点が把握できる構成にする |
| 9 | 公式文書としての作法を守る | 広告ではなく公式文書。客観的事実を正確かつ端的に記載 |
| 10 | ニュースバリューを明確にする | 「他社と比べて特筆すべき点」「社会的意義」を明示 |
| 11 | 感嘆符・絵文字・砕けた表現は避ける | 企業としての信頼を損なわないよう |
| 12 | 参考資料をPDF等で添付する | 本旨と関連性の低い詳細情報は別紙に |

### 1-3. タイトルの書き方の詳細ルール

PR TIMES MAGAZINEのタイトル解説記事から抽出した具体的なルールは以下の通りである。

- **文字数**: 1行あたり25〜30字未満が理想。全体で41〜100字前後
- **重要キーワードは前半に配置**: 人間がひと目で把握できる文字数は13文字。前半に核心を置く
- **数字・固有名詞を入れて具体的に**: 「売上〇〇個達成」「営業利益〇％アップ」など
- **ニュースバリューを意識**: 「日本初」「業界最大」等は根拠を明示（自社調べでも可）
- **墨カッコ【】の活用**: 【新発売】【調査レポート】【緊急支援】等で目を引く
- **サブタイトルの活用**: タイトルが長くなる場合、補足情報をサブタイトルに分離

**良いタイトルの事例**:
- 「【新発売】ブランド累計販売66万食『Fun Fiber』人気のグラノーラの新味『アップルグラノーラ』」
- 「【女性の労働に関する調査】過去4年で月経における労働損失・欠勤率が改善」
- 「【緊急支援】〇〇被害へ寄付の募集を開始：水や食料援助のお願い」

### 1-4. リード文の書き方の詳細ルール

- **文字数**: 250〜300字程度が理想。最大400字以内
- **構成**: 5W2H＋展望で網羅する
- **冒頭の型**: 「〇〇株式会社（本社：〇〇、代表取締役：〇〇）は、〇〇を〇月〇日より開始いたします。」
- **上場企業の場合**: 市場区分とコードを加える
- **専門用語は不使用**: 一般の方でも分かる表現に
- **展望を加える**: 企業の思いや今後の方向性を1文添えると効果的

### 1-5. 本文の書き方の詳細ルール

- **結論先行**: 通常の起承転結ではなく「結起承転」が基本
- **逆三角形構造**: 最重要情報→背景→詳細の順
- **1プレスリリース1テーマ**: A4サイズ1〜3枚が目安
- **太字・下線で強調**: やりすぎない程度に
- **表や箇条書き**: 複雑な情報の整理に活用
- **過去実績等は後半に**: 発表の本旨と関連性の低い内容は後ろへ

### 1-6. メディア編集長の視点（インプレス社解説より）

PR TIMES MAGAZINEにインプレス社メディア編集長が寄稿した記事によると、メディア側がプレスリリースを判断する際のポイントは以下の通りである。

- **ファーストビュー（タイトル・画像・冒頭文）が勝負**: 1通あたりにかける時間は2〜3秒
- **タイトルと画像で記事候補になるかを瞬時に判断**
- **数字のインパクト**: 調達額、利用者数、前年比などの具体的な数値が決め手になる
- **社会的文脈との接続**: なぜ今このニュースが重要なのかが明示されていること

## 3. カテゴリ横断の構成パターン分析

### 3-1. タイトルの書き方パターン

調査した事例からタイトルの書き方を類型化すると、以下のパターンが見えてくる。

**パターンA: 社名＋アクション＋数値**（資金調達に多い）
- 「EX-Fusion、シリーズAで総額約26億円の資金調達を実施」（24文字）
- 「Space Aviation、シリーズBで17.4億円の資金調達を実施」（23文字）
- 最も簡潔で、メディアがそのまま見出しにしやすい。

**パターンB: 数値実績＋サービス名＋アクション**（サービスリリースに多い）
- 「多拠点型施設運営SaaS『knotPLACE』利用者5.5万人突破・サービスサイトのフルリニューアルを実施」（35文字）
- 実績数値で関心を引き、何のサービスか→何が起きたかの順に伝達。

**パターンC: 【墨カッコ】＋インパクトワード＋詳細**（イベント・新商品に多い）
- 「【開催決定】THE SNS CONFERENCE 2025｜最前線の実践者たちが語る...」
- 「【新発売】ブランド累計販売66万食『Fun Fiber』...」
- 墨カッコで種別を即座に伝え、後半で内容を補足。

**パターンD: イベント名＋規模感数値＋日程**（カンファレンスに多い）
- 「国内最大級のAIエージェントカンファレンス...総勢41名の登壇者、30を越えるセッション...」
- 数字の羅列で大規模さを印象付ける。

**タイトル文字数の傾向**:
- 資金調達: 23〜24文字（極めて簡潔）
- サービスリリース: 35〜81文字（幅が広い）
- イベント開催: 55〜75文字（情報量が多い傾向）
- 業務提携: 40〜66文字（両社名を含むため長くなる傾向）

### 3-2. サブタイトルの活用パターン

調査した10事例のうち、サブタイトルを使用していたのは5事例。以下の場合に特に効果的である。

- **資金調達の使途を補足**: 「M&Aによる海外展開を推進」（Space Aviation）
- **ビジョンを伝達**: 「〜テクノロジーとサポートでクラウド時代のSaaS管理体制を構築〜」（ジョーシス）
- **権威性を付与**: 「Apple子会社Clarisが日本で開催する最大のイベント...」（Claris）
- **登壇者名でインパクト**: 「清水建設、ライオン、デロイトトーマツ...」（AI Agent Day）

### 3-3. リード文の構造パターン

リード文の文字数は事例により100〜340字と幅があるが、共通する構造がある。

**基本構造**:
```
[会社名]（本社：[所在地]、代表取締役：[氏名]）は、
[何を]（発表内容の端的な記述）
[いつ]から[どこで]
[実施/開始/完了]したことをお知らせいたします。
[補足情報（金額・規模等）]
```

**優良事例の共通点**:
- 冒頭で「誰が」「何を」を明示
- 金額や数値は必ず含める
- 最後に展望や意義を1文添える
- 専門用語は使わない

### 3-4. 本文の見出し構成パターン

**資金調達のプレスリリース**（標準構成）:
1. 背景/目的
2. 資金調達の内容（金額内訳、出資者一覧）
3. 投資家からのコメント
4. 代表コメント
5. 今後の展望
6. 会社概要

**サービスリリースのプレスリリース**（標準構成）:
1. サービス概要（何ができるか）
2. 開発の背景/提供の背景
3. 主な機能・特徴
4. 料金プラン/導入方法
5. 今後の展望
6. 会社概要

**業務提携のプレスリリース**（標準構成）:
1. 提携の背景（社会課題・市場動向）
2. 提携の目的と概要
3. 各社の役割
4. 関係者コメント
5. 今後の展望
6. 各社の会社概要

**イベント開催のプレスリリース**（標準構成）:
1. 開催概要（日時・場所・参加費）
2. イベントの特徴/見どころ
3. プログラム/セッション一覧
4. 登壇者紹介
5. 申込方法
6. 主催者情報

### 3-5. 数値・データの使い方

調査した優良事例に共通する数値の使い方の特徴は以下の通りである。

- **タイトルに必ず1つ以上の数値を入れる**: 金額、人数、件数、日数のいずれか
- **リード文に累計・実績数値を含める**: 累計調達額、導入企業数、利用者数
- **前年比・成長率で変化を可視化**: 「前年比57倍」「10倍拡大」
- **技術的なマイルストーンも数値化**: 「1秒間に10回の核融合反応」
- **公的データの引用で客観性を確保**: 「経済産業省『DX推進指標』」「最大12兆円/年の経済損失」
- **将来の目標も数値で示す**: 「2027年度」「200名増員」「2030年頃」

### 3-6. 代表コメントの書き方

代表コメントの掲載パターンは以下の3つに分類される。

1. **代表者コメントのみ**: 経営ビジョン、事業方針、社会的意義を語る
2. **投資家/パートナーコメントのみ**: 第三者からの評価として信頼性を担保（EX-Fusionの例）
3. **代表者＋投資家/パートナーの両方**: 最も効果的。多角的な視点を提供（Space Aviationの例）

**コメントの書き方のコツ**:
- 2〜3段落で簡潔に
- 個人的な熱意よりも事業の社会的意義を中心に
- 具体的な将来計画を含める
- 読み手（メディア・投資家・ユーザー）を意識した言葉選び

### 3-7. 会社概要セクションの構成

すべての優良事例で共通して含まれていた項目は以下の通りである。

- 会社名（正式名称）
- 本社所在地
- 代表者名（役職付き）
- 設立年月
- 事業内容
- WebサイトURL

加えて、以下の項目を含む事例が多かった。

- 資本金
- 上場状況（上場企業の場合は証券コード）
- 電話番号
- サービスサイトURL（コーポレートサイトとは別に）
- 業種

### 3-8. 画像の使い方

調査した事例の画像枚数は1〜20枚と幅が大きい。カテゴリ別の傾向は以下の通りである。

| カテゴリ | 平均枚数 | 画像の種類 |
|----------|----------|------------|
| 資金調達 | 1〜4枚 | 技術画像、代表者顔写真、投資家顔写真、概要図解 |
| サービスリリース | 4〜5枚 | サービス画面、機能説明図、導入ステップ図、利用シーン |
| 業務提携 | 2枚 | 企業ロゴ、提携イメージ図 |
| イベント開催 | 2〜20枚 | メインビジュアル、登壇者写真、タイムテーブル、会場写真 |

**画像で特に効果的だったもの**:
- **代表者・関係者の顔写真**: メディアが記事に使いやすく、取材依頼の判断材料になる
- **サービス概要の図解**: 複雑なサービスの価値を一目で伝える
- **数値を含むインフォグラフィック**: 資金調達の内訳や成長推移のグラフ
- **メインビジュアル**: SNSシェア時のOGP画像として最も重要

## 4. 特に注目すべきテクニック

### 4-1. メディアが記事にしやすい書き方

調査した優良事例とPR TIMES公式ガイドラインから抽出した、メディアが記事にしやすい書き方のテクニックを以下にまとめる。

1. **タイトルをそのまま記事見出しに使える簡潔さ**: 特に資金調達のプレスリリースは23〜24文字で完結しており、メディアがそのままコピーして使える。

2. **「結起承転」の逆三角形構造**: 最重要情報がタイトル→リード文で完結するため、メディア記者がリード文まで読むだけで記事の骨格を把握できる。

3. **公的データ・第三者データの引用**: 経済産業省のデータや業界統計を引用することで、記者が独自に裏取りする手間を省いている。

4. **顔写真の提供**: 代表者や関係者の顔写真を添付することで、記事に使う素材を提供。メディア側の取材コストを下げる効果がある。

5. **投資家・パートナーのコメント**: 第三者のコメントはそのまま記事に引用でき、記事の客観性を確保する素材となる。

6. **図解・インフォグラフィック**: 複雑なビジネスモデルや資金の流れを図解で提供することで、メディアが記事に画像を挿入しやすくなる。

### 4-2. SNSでシェアされやすいタイトルの特徴

1. **具体的な数値のインパクト**: 「140億円」「56億円」「41名の登壇者」など、桁が大きい数字や意外性のある数字はSNSで注目を集めやすい。

2. **【墨カッコ】による分類**: 【開催決定】【新発売】【調査レポート】等の墨カッコは、タイムライン上で目を引き、内容の種別が即座に伝わる。

3. **トレンドキーワードの含有**: 「AI」「AIエージェント」「DX」「SaaS」「ローコード」など、その時期に検索されやすいキーワードを含める。2025年は「AIエージェント」が前年比57倍の急成長。

4. **問いかけ型のフレーズ**: 「"これからのSNSとの向き合い方"」のような問いかけは、読み手の共感を誘い、SNSでのコメント付きシェアを促進する。

5. **有名企業名の列挙**: サブタイトルに「清水建設、ライオン、NVIDIA」等の大手企業名を並べることで、権威性とシェアのインセンティブを生む。

### 4-3. 検索で見つかりやすい工夫（SEO観点）

1. **タイトルの前半13文字に主要キーワードを配置**: 検索結果で表示されるタイトル文字数は約30文字。前半に核心キーワードを置くことで、検索結果でのクリック率が向上する。

2. **サービス名・イベント名をカギカッコで囲む**: 「knotPLACE」「AI Agent Day 2025 Summer」等の固有名詞をカギカッコで囲むことで、検索エンジンが固有名詞として認識しやすくなる。

3. **業界キーワードの自然な含有**: 「SaaS」「資金調達」「業務提携」「カンファレンス」など、検索されやすいカテゴリキーワードを本文中に自然に含める。

4. **数値をアラビア数字で表記**: 「26億円」「17.4億円」「5.5万人」など、数値はアラビア数字で表記することで検索エンジンが数値として認識しやすい。

5. **会社名の正式名称と略称の両方を含める**: リード文で「株式会社〇〇（以下『〇〇』）」と記載することで、正式名称でも略称でも検索にヒットする。

## 5. 調査まとめ -- プレスリリース作成のチェックリスト

本調査の結果から、高い反響を得るプレスリリースを作成するためのチェックリストを以下に整理する。

### タイトル
- [ ] 30文字以内で結論が伝わるか
- [ ] 数値が少なくとも1つ含まれているか
- [ ] 重要キーワードが前半13文字以内に配置されているか
- [ ] 感嘆符・絵文字は使っていないか
- [ ] サブタイトルで補足情報を分離しているか（必要な場合）

### リード文
- [ ] 250〜300字（最大400字以内）に収まっているか
- [ ] 5W2H（Who/What/Where/When/Why/How/How much）が網羅されているか
- [ ] 「展望」が1文含まれているか
- [ ] 専門用語は使っていないか

### 本文
- [ ] 結論先行の逆三角形構造か
- [ ] 1テーマに絞られているか
- [ ] 小見出しで構造化されているか
- [ ] 数値・データで裏付けがあるか
- [ ] 社会的背景・トレンドとの接続があるか

### コメント
- [ ] 代表者または投資家/パートナーのコメントが含まれているか
- [ ] コメントは2〜3段落で簡潔か

### 画像
- [ ] 3〜5枚以上掲載されているか
- [ ] メインビジュアル（OGP用）が用意されているか
- [ ] 代表者・関係者の顔写真があるか
- [ ] サービス・製品の図解があるか

### 会社概要・連絡先
- [ ] 会社名・所在地・代表者・設立年月・事業内容・URLが含まれているか
- [ ] 担当部署・担当者名・電話番号・メールアドレスが記載されているか
